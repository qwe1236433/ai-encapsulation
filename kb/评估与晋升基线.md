# 评估与晋升基线（一步一步）

> **原则**：**评估**只产生指标与 artifact；**晋升**才替换工厂读入的 JSON。避免单次 hold-out 噪声直接改生产。

## 1. 合并 Feed（带质量报告）

在仓库根目录 PowerShell：

```text
.\scripts\merge-xhs-feed.ps1
```

- 默认 **`validate=report`**（未配置 `FLOW_API_EXPORT_VALIDATE_MODE` 时）：stderr 出现 `validate_stats`，**仍写出** `samples.json`。  
- 关闭校验：`.\scripts\merge-xhs-feed.ps1 -ValidateMode none`或在 `.env` 设 `FLOW_API_EXPORT_VALIDATE_MODE=none`。  
- 硬门禁（不写失败条目的 out）：在 `.env` 设 `FLOW_API_EXPORT_VALIDATE_MODE=fail`或传 `-ValidateMode fail`（有违规时 **不** 覆盖 `samples.json`）。
- **入库前健康度**（条数、时间列可解析率/跨度、按 `labels_spec` 的正例数等）：未配置 `FLOW_API_FEED_HEALTH_GATE_MODE` 时 **`merge-xhs-feed.ps1` 默认 `report`**，写入 **`research/runtime/feed_ingest_health.json`**，不挡写出。硬拦截：设 **`FLOW_API_FEED_HEALTH_GATE_MODE=fail`** 并提供 **`FLOW_API_FEED_HEALTH_SPEC`**（模板 `scripts/feed_ingest_health.example.json`）；未过门禁时与 validate fail 一样**不写** `samples.json`。

计划任务若使用 **`merge-xhs-feed-with-log.ps1`**，行为与上相同，日志在 `logs\merge-xhs-feed.log`。

### 1.1 排查用语与互动口径（微调）

- **小样本 / 负例极少**：分层或 hold-out 下更容易出现**某一折只有单一类别、AUC 不可定义或剧烈跳变**——这是**有效样本量与划分方差**问题。不宜笼统称为「数据泄漏」；**泄漏**一般指训练与测试间**错误共享信息**（如同一条目重复、时间顺序颠倒、使用未来才可得标签等），需单独举证。  
- **`like_proxy` 与「是否填充」**：合并脚本 `scripts/export_to_xhs_feed.py` 在归一时，若原始行**缺少**点赞类字段，会为 `like_proxy` 赋**默认正整数**（与爬虫真实低赞需在原始 jsonl 对照区分）。排查时先看原始行是否缺 `liked_count` / `likes` 等键。

## 2. 导出特征并校验契约

```text
python scripts\export_features_v0.py --samples openclaw\data\xhs-feed\samples.json --out research\features_v0.csv --labels-spec research\labels_spec.json
```

（若已有 digest：加 `--feed-digest` 与 `--verify-samples-digest`。）

```text
python scripts\verify_features_labels_spec.py --features research\features_v0.csv --labels-spec research\labels_spec.json
```

默认会在 **`research/runtime/features_export_provenance.json`** 写入本次导出的 **samples sha256、digest、batch_id、行数** 与对外陈述提醒；若不需要可加 **`--no-provenance`**。

**一键串线（单次跑完 merge → 关键词轮换 → 导出 → 校验 → 指标 → 训练 → 评估 → 归档 → 对比报告）**：

```text
.\scripts\xhs-research-oneclick.ps1
```

轮换词表：复制 **`research/keyword_rotation_pool.example.txt`** 为 **`research/keyword_rotation_pool.txt`** 后编辑（已在 `.gitignore`）。对比报告：**`research/runtime/digest_comparison_report.md`**（亦由 **`python scripts/compare_digest_analytics_runs.py --last 3 --include-current`** 单独生成）。

### 2.1 对外陈述与数据防线（结论怎么写才不虚）

- **数据范围**必须可指回物化证据：本表对应的 **`samples.json` 路径 +文件 sha256**；若用了 digest，写 **`samples.digest.json` 内 `sha256` 与 `batch_id`**（与 `features_v0.csv` 列一致）。  
- **单批次快照**：`batch_id` 与 `feed_digest_sha256` 在表内**仅一种取值**时，结论只能是**该次合并快照内的共变/校准**，不得写成「全站」「任意批次」泛化。  
- **互动口径**：合并已解析常见 **`万` / `w`** 点赞简写（`openclaw/feed_like_parse.py`）；若仍见大量 `like_proxy=100`，需对照原始 jsonl 是否缺字段或真为低赞。  
- **机器可读防线**：`python scripts/compute_feed_metrics_v0.py` 产出的 **`warnings`**（单批次、时间解析率低、`like_proxy` 大量为 100 等）应进实验记录或 CI 日志。  
- **可直接粘贴的摘要句**（按实际替换占位）：  
  - 「本结果基于 **N=（行数）** 条特征行，对应 **`samples.json` sha256=（前16位…）**、digest **`batch_id`=（值）**；**不**声称跨批次或全站外推。」

## 3. 评估（仅记录，不部署）

**方式 A — 命令行**

```text
python research\train_baseline_v0.py --features research\features_v0.csv --out research\artifacts\eval_candidate.json --labels-spec research\labels_spec.json --cv-folds 5
```

**方式 B — Flow API**（需已启动 `api/main.py`）

- `POST http://127.0.0.1:8099/api/model/evaluate`  
- Body 可用默认：`features_path`、`labels_spec` 省略时使用 `research/features_v0.csv` 与 `research/labels_spec.json`。  
- 响应含 `holdout_roc_auc`、`holdout_brier_score`、`cross_validation`、`input_features_sha256`、`artifact_path` 等。

**本步不做**：修改 `XHS_FACTORY_BASELINE_JSON`、重启容器。

## 4. 人工决策

- 把指标记入 `research/EXPERIMENT_REPORT.md`或你们内部实验编号。  
- 对比上一版 artifact（若存在）：样本量、`input_features_sha256`、批次是否一致。

## 5. 晋升（受控复制）

```text
.\scripts\promote_baseline.ps1 -Source research\artifacts\eval_candidate.json -Target openclaw\data\artifacts\baseline_v0.json -BackupDir openclaw\data\artifacts\backups
```

- 目标已存在且未设 `-BackupDir`时，需加 **`-AllowOverwrite`** 才会覆盖（或始终使用 `-BackupDir`）。  
- 在 `.env` / `docker-compose` 中确认 **`XHS_FACTORY_BASELINE_JSON`** 指向容器内可读路径（与 `OPENCLAW_DATA_DIR` 挂载一致），必要时 **`docker compose up -d`** 或重启 openclaw。

## 6. 与 Flow API合并行为的对照

| 入口 | 未配置 `FLOW_API_EXPORT_VALIDATE_MODE` 时 |
|------|-------------------------------------------|
| **`merge-xhs-feed.ps1`** |默认 **`report`**（脚本内置默认） |
| **`POST /api/export-feed`**（`api/main.py`） |读进程环境：无则 **`none`**；**`.env.example` 已默认写入 `FLOW_API_EXPORT_VALIDATE_MODE=report`**，复制到 `.env` 后与 merge 对齐 |

若你从旧 `.env` 升级且未包含该键：**merge 仍有 report**，**Flow API 合并无校验**，直到你在 `.env` 补上该变量。
