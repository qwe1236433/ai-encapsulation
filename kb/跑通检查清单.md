# 跑通检查清单（按顺序做，做完打勾）

> 环境：**Windows**，仓库根目录 **`D:\ai封装`**。PowerShell 里跑脚本必须带 **`.\`**。  
> 用途：自检「从数据到一篇导出 txt」是否贯通；卡住时看对应文档。

---

## A. 环境与文件- [ ] **A1** 已安装 Docker Desktop，且能正常启动。  
- [ ] **A2** 仓库根目录存在 **`.env`**（对照 `.env.example`）。  
- [ ] **A3** 已确认爬虫 jsonl 路径存在，或暂时用工厂 mock（见 `使用说明书`）。  
- [ ] **A4** 目标 Feed 文件存在或即将生成： `openclaw\data\xhs-feed\samples.json`（可先跑合并脚本再检查）。

---

## B. 合并 Feed（爬虫 → samples.json）

在 **`D:\ai封装`** 打开 PowerShell：

- [ ] **B1** 执行（路径按你本机改 `--in`；需要去重时加 `--dedupe key` 或先设 `FLOW_API_EXPORT_DEDUPE`）：  
 `python scripts\export_to_xhs_feed.py --in D:\MediaCrawler\data\xhs\jsonl --out openclaw\data\xhs-feed\samples.json`  
     或：`.\scripts\merge-xhs-feed.ps1`（从 `.env` 读路径与去重策略）  
- [ ] **B2** 打开 `samples.json` 看一眼：是数组、条数 > 0、字段合理。

---

## C. Docker 与 OpenClaw / Hermes

在 **`D:\ai封装`**：

- [ ] **C1** `docker compose up -d` 成功（或你用惯用的等价命令）。  
- [ ] **C2** 浏览器能打开：`http://127.0.0.1:8080/`（Hermes）、`http://127.0.0.1:3000/`（OpenClaw）。

---

## D. 生成文案（二选一或都做）

**方式 1：命令行 bench（推荐先会这个）**

- [ ] **D1** 在仓库根执行：  
      `.\bench-hermes-xhs-sync.ps1`  
      （不要写成 `bench-hermes-xhs-sync.ps1` 不带 `.\`）  
- [ ] **D2** `outputs\xhs-runs\` 下出现新的 `.txt`，打开有 headline/body 等内容（goal 需含小红书语境，见说明书）。

**方式 2：网页一键（Flow API）**

- [ ] **D3** `.\scripts\start-flow-api.ps1` 无报错。  
- [ ] **D4** 浏览器 `http://127.0.0.1:8099/` 点「一键开始」，日志跑完。  
- [ ] **D5** 同样检查 `outputs\xhs-runs\` 或页面「最新导出」。

---

## E. 研究侧（可选，与 Wiki 并行）

- [ ] **E0**（推荐）复制 `research\labels_spec.example.json` → `research\labels_spec.json`，按本批次改阈值与说明。  
- [ ] **E1** `python scripts\export_features_v0.py --samples openclaw\data\xhs-feed\samples.json --out research\features_v0.csv --labels-spec research\labels_spec.json`（或继续用 `--viral-threshold 1000`）  
- [ ] **E2**（可选）安装 `research\requirements-research.txt` 后跑 `research\train_baseline_v0.py`，得到 `research\artifacts\baseline_v0.json`。

---

## F. 卡住时查谁

| 现象 | 先看 |
|------|------|
| 脚本「找不到命令」 | `使用说明书.md` 里 PowerShell `.\` 说明 |
| goal 不对、只有 mock 流量 | `使用说明书` / bench 里「小红书」前缀与 Hermes 路由 |
| 容器读不到样本 | `数据流与文件落点.md` 第 3 节挂载 |
| 数字结论能不能写进 Wiki | `research/EXPERIMENT_REPORT.md` 去幻觉节 |

---

**使用方式**：复制本页到你的笔记里打勾也行；核心是 **A→B→C→D** 顺序不要跳。
