# OpenClaw + Hermes 小红书文案流水线 — 使用说明书

面向：**在 Windows 上已能 `docker compose up` 跑起来**的你。尽量白话，少术语。

**完全新手、听不懂「爬虫怎么和工厂连起来」**：先看根目录 **`新手请看-工厂和爬虫怎么配合.md`**（用打比方 + 5 步操作，不堆术语）。

---

## 这套东西是干什么的？

- **Hermes**：大脑调度，按步骤调用「挖掘 → 二创 → 预测 → 整理成待发笔记」。
- **OpenClaw**：干活容器，里面会调 **MiniMax** 写标题/正文（密钥配在 `.env` 里）。
- **小红书**：程序**不会**自动登录发帖；**只会生成文案**，你要自己复制到 App 里发。

默认未配置样本路径时，工厂用 **模拟样本**；若要在本地接入 **MediaCrawler 等导出数据**，先看 **`爬虫与工厂-对接清单与双文件夹打开步骤.md`**（勾选什么算合格、Cursor 怎么开双文件夹），需要架构说明时再看 **`小红书数据与工厂-架构技术指南.md`**。

---

## 开始前你要有什么？

1. 已安装 **Docker Desktop**，本项目目录下能执行：
   ```text
   docker compose up -d
   ```
2. 根目录有 **`.env`**（可从 `.env.example` 对照填写），至少包括：
   - **MiniMax**：`MINIMAX_API_KEY`、`MINIMAX_BASE_URL`（国内 Key 常用 `https://api.minimaxi.com/v1/text/chatcompletion_v2`）
   - **会话目录**：`HERMES_SESSIONS_DIR` 指到你本机路径（与 compose 挂载一致）
3. 浏览器能打开：
   - Hermes：`http://127.0.0.1:8080/`
   - OpenClaw：`http://127.0.0.1:3000/`（可看 `minimax_key_ok` 是否为真）

改完 `.env` 后执行一次 **`docker compose up -d`**，不必每次跑 bench 都改。

## 本机流程控制台（`api/` + `web/`）

**`api/main.py`**（FastAPI）提供 **`/api/health`**、**合并 Feed**、**docker compose up**、**异步 Bench**、**最新导出预览**；**`web/`** 为同步操作界面（调用上述接口）。

```powershell
Set-Location -LiteralPath "D:\ai封装"
.\scripts\start-flow-api.ps1
```

浏览器打开 **`http://127.0.0.1:8099/`**；API 说明 **`http://127.0.0.1:8099/docs`**。**Ctrl+C** 结束。仅绑定本机，勿对公网暴露。

页面上 **「一键开始」** 会依次调用接口 **`POST /api/run/full-pipeline`**：合并爬虫 JSONL → Feed（`samples.json`）→ **`docker compose up -d`** → **`bench-hermes-xhs-sync.ps1`**。**goal** 即**本次想写的主题与要求**：在网页大文本框里每次填写即可（请求体字段 **`goal_text`**）；若留空则用 **`goal_path`** 指向的 UTF-8 文件，再空则用 **`FLOW_API_BENCH_GOAL`** 默认文件。日志区会显示阶段说明；若你已有维护好的 `samples.json`，可勾选 **跳过合并**。分步按钮收在 **「高级：分步执行」** 里。

可选环境变量见 **`.env.example`** 中 `FLOW_API_*`（爬虫路径、Feed 输出、goal 默认等）。

`serve-flow-page.ps1` 已等同于启动本控制台（内部转调 `start-flow-api.ps1`）。

## 一键生成文案（推荐）

**路由说明**：Hermes 只有判断 goal 与 **小红书 / xhs** 相关时才会走 **XHS 工厂链**（`extract_viral_patterns` → … → **`prepare_xhs_post`**，才有标题正文）。若 goal 里完全没有这些词，可能只跑到 **`predict_traffic`** 等通用算子，导出 txt 里 **`notes` 会是 `mock predict_traffic`**，标题正文为空。**`bench-hermes-xhs-sync.ps1` 已自动处理**：若你的 goal 里尚未包含「小红书」「小红薯」或 `xhs`，脚本会在发给 Hermes 前加上前缀 **`小红书：`**，避免误走流量 mock。

在项目根目录 `D:\ai封装` 打开 **PowerShell**：

**PowerShell 必看**：运行当前文件夹里的脚本时，名字前要加 **`.\`**，例如 **`.\bench-hermes-xhs-sync.ps1`**。若只输入 `bench-hermes-xhs-sync.ps1`，会提示找不到命令（当前目录不在 PATH 里，这是正常行为）。

```powershell
Set-Location -LiteralPath "D:\ai封装"
.\bench-hermes-xhs-sync.ps1
```

自定义主题（中文也可以）：

```powershell
.\bench-hermes-xhs-sync.ps1 -Goal "在小红书发一篇副业复盘笔记，要爆款标题" -MaxAttempts 6
```

**长中文 goal（推荐）**：把目标写进 **UTF-8** 的 `.txt`，用 `-GoalPath` 读入，可避免 PowerShell 控制台编码把参数弄乱：

```powershell
.\bench-hermes-xhs-sync.ps1 -GoalPath ".\scripts\bench-goal-example.txt" -MaxAttempts 6
```

**只打印 JSON、不写 txt：**

```powershell
.\bench-hermes-xhs-sync.ps1 -NoExport
```

## 生成的文章输出到哪里？（重点）

### 1）自动导出的 TXT（本次专门为你做的）

每次跑 **`bench-hermes-xhs-sync.ps1`**（且未加 `-NoExport`）后，会生成：

| 位置 | 说明 |
|------|------|
| `outputs\xhs-runs\` | 每次一篇**单独**的 `.txt`，文件名带**时间 + 任务 ID 前 8 位** |
| `outputs\xhs-articles-log.txt` | **汇总日志**，每次运行**追加**一段，方便翻历史 |

**说明**：仓库里已带好空目录 `outputs\xhs-runs\`（里面有 `.gitkeep`）。若你本地曾经没有，多半是还没用**新版** bench 成功跑完一次，或用了 `-NoExport`。跑一次 `.\bench-hermes-xhs-sync.ps1` 后会出现 `.txt` 文件。详见 `outputs\README.txt`。

文件编码为 **UTF-8（带 BOM）**，Windows 记事本也能正常打开中文。

TXT 里一般包含：**标题、正文、话题标签、配图描述、备注**，以及 **任务 ID** 和 **查询 JSON 用的网址**。

### 2）接口里的 JSON（和 TXT 是同一份内容）

把里面的 `task_id` 换成你当次返回的 UUID：

```text
http://127.0.0.1:8080/task/你的task_id
```

在 JSON 里找：**`final_envelope` → `openclaw` → `result`**，里面有 `headline`、`body`、`hashtags` 等。

### 3）会话文件（若已挂载到本机）

路径在返回 JSON 的 **`session_file`** 字段；内容也是整次任务快照（JSON）。

## 发帖之后要做什么？（可选）

1. 在小红书 **手动发布**。
2. 在 App 里找到该笔记的 **笔记 ID**（`real_note_id`）。
3. 在项目根目录执行：

```powershell
.\bench-xhs-post-sync.ps1 -TaskId "你的task_id" -RealNoteId "真实笔记ID"
```

之后 Hermes 会按当前设计走 **模拟拉数**（非真实平台 API）；真数据需以后改代码对接。

## 常见问题（白话）

**问：`minimax_ready`  true 但工厂还是像模板？**  
看 OpenClaw 首页的 **`minimax_key_ok`**。只有它为 true，才说明 Key 被官方接受。

**问：还要不要每次动 `.env`？**  
不用。只有改密钥、改模型、改地址时才改，然后 `docker compose up -d`。

**问：扒小红书爆款帖分析有没有？**  
**没有。** 现在「挖掘」用的是程序里生成的假样本；真爬取/官方 API 要以后做在 `openclaw/xhs_factory.py` 的 `_fetch` 等处。

**问：`outputs` 能提交 Git 吗？**  
默认已在 **`.gitignore`** 里忽略，避免把生成文案误传到仓库。

## 提示词在哪里改？（外置 YAML）

小红书工厂 **MiniMax 提示词**与 **`prepare_xhs_post` 本地模板**已放到：

`openclaw/prompts/xhs/`

| 文件 | 用途 |
|------|------|
| `extract_viral_patterns.yaml` | 挖掘结构（system + user 模板，`$topic` 等） |
| `recreate_content.yaml` | 二创正文（system + `limits.original_max_chars`） |
| `predict_viral_score.yaml` | 预测分（system + 文本/案例库长度上限） |
| `prepare_xhs_post.yaml` | 待发清单：缺标题/正文时的模板、封面描述、checklist 文案 |

改完 YAML 后请 **`docker compose build openclaw --no-cache`** 再 **`docker compose up -d`**，避免层缓存仍是旧提示词（镜像内 `COPY prompts`）。  
若你本机 `docker compose build` 报 `unknown flag: --no-deps`，可忽略带 `--no-deps` 的写法，只保留 **`--no-cache`** 即可。  
可选：设置 **`OPENCLAW_PROMPTS_DIR`** 指向自定义目录（需含同名 `.yaml`）。详见 `openclaw/prompts/xhs/README.txt`。

## 生成的 txt 里中文变成乱码时（按顺序试，不用懂术语）

**正常长什么样**：打开 `outputs\xhs-runs\` 里的某个 `.txt`，里面是通顺的中文标题和正文。  
**不正常长什么样**：满屏像 `å�� ç�` 这种看不懂的符号。

下面三件事任选一种能做到就行。

### 办法一：先保证是用「整个项目文件夹」打开的（最常解决问题）

1. 关掉 Cursor。  
2. 再打开 Cursor。  
3. 点顶部菜单 **文件** → **打开文件夹**（英文界面是 **File** → **Open Folder**）。  
4. 只选 **`D:\ai封装` 这个文件夹**，点确定。**不要**只单独打开某一个 `.txt` 文件。  
5. 在左边文件树里依次点开：`outputs` → `xhs-runs` → 再点开你的 `.txt`。

项目里已经放好 **`.vscode\settings.json`**，用「打开文件夹」的方式进来，一般会按正确方式显示中文。

### 办法二：只对「这一篇」乱码的 txt 动手

1. 在 Cursor 里打开那篇 txt。  
2. 看窗口 **最右下角**（紧贴着底边），有一小段字，可能是 **UTF-8**、**GBK**、**Western** 等；**用鼠标左键点一下** 这段字。  
3. 点 **通过编码重新打开**（英文是 **Reopen with Encoding**）。  
4. 在列表里选 **UTF-8**；若列表里有 **UTF-8 with BOM**，也可以选它。  
5. 看正文是否变回中文。变回来就成功了。

### 办法三：不用 Cursor，用 Windows 记事本看

1. 在资源管理器里找到那个 `.txt`，**右键** → **打开方式** → **记事本**。  
2. 多数情况下中文会直接正常。若仍乱：记事本里点 **文件** → **另存为**，窗口最下面 **编码** 选 **UTF-8**，保存（会覆盖原文件，一般不必走到这一步）。

### 办法四：上面都做了还是乱码（重要）

**原因**：旧版 `bench-hermes-xhs-sync.ps1` 用 PowerShell 自带的 **`Invoke-RestMethod`** 读接口，在中文版 Windows 上容易把 **JSON 里的中文先读坏**，再写进 txt，所以你怎么改编辑器编码都没用。

**已改好**：bench 已改为用本仓库的 **`utf8-http.ps1`**（按 UTF-8 读网络返回），**请再跑一次**：

```powershell
Set-Location -LiteralPath "D:\ai封装"
.\bench-hermes-xhs-sync.ps1 -Goal "XHS note: your topic" -MaxAttempts 6
```

**看新生成的** `outputs\xhs-runs\` 里最晚时间戳的那个 `.txt`。**旧的那几份 txt** 里的字已经坏在文件里了，**没法靠换编码救回来**，可以忽略或删掉。
