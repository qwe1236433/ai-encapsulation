# 新手请看：工厂和爬虫怎么配合（尽量白话）

如果你暂时听不懂「架构」「工作区」「JSON 契约」也没关系，**只看这一篇**就够入门。

---

## 打个比方

- **`ai封装` 这套程序**：像一台 **做菜机器**。你给它一个主题，它帮你写出小红书文案。  
- **爬虫（比如 MediaCrawler）**：像 **去菜场买菜**。它从网上把别人的笔记标题、正文、点赞数 **抄下来**，变成电脑里的 **文本文件**。  
- **你要做的**：让「买菜」的结果，变成做菜机器 **认得出的一袋菜**（一个固定位置的文件）。机器 **不会** 自动去你爬虫里拿菜，**只认你放在指定抽屉里的那份文件**。

所以：**不用**把爬虫和 `ai封装` 合成一个项目；**只要**最后有一个文件放对地方、格式大致对就行。

---

## 你只要记住 3 件事

1. **爬虫跑完后**，会生成一些数据文件（可能是 `.json` 之类）。  
2. **你要让这些内容**，最后变成 **`ai封装` 里某一个文件夹里的一个文件**（我们约定好放哪里）。  
3. **在 `.env` 里写一行字**，告诉程序：「请去读那个文件」。然后 **重启一下 Docker 里的服务**（如果你是用 Docker 跑的）。

听不懂「.env」「Docker」也没关系：下面有「第一次怎么做」按顺序做。

---

## 第一次建议你怎么做（按顺序）

### 第 1 步：先别管爬虫，确认机器能转

按 `使用说明书.md` 里写的，用 Docker 把 `ai封装` 跑起来，能打开网页、能跑一键脚本生成文案。  
这一步是确认 **做菜机器本身能正常运转**。

### 第 2 步：准备一个「示例菜」文件（不用爬虫也能试）

1. 打开文件夹：`D:\ai封装\openclaw\data\xhs-feed\`  
2. 里面有一个 **`samples.example.json`**。  
3. **复制**一份，改名为 **`samples.json`**（和 example 并排放着就行）。

这个文件里已经是工厂能读的格式，用来 **假装**是爬虫采来的数据。

**仓库里还有一份「牛排熟度」实测样本**（多条笔记拼成）：`openclaw/data/xhs-feed/samples-steak-test.json`。把 `.env` 里的 `XHS_FACTORY_SAMPLES_PATH` 改成指向这个文件，即可用牛排主题跑一轮工厂测试。

### 第 3 步：告诉程序去读这个文件

1. 打开 **`D:\ai封装\.env`**（如果没有，从 `.env.example` 复制一份改名成 `.env`）。  
2. 找有没有 **`XHS_FACTORY_SAMPLES_PATH`** 这一行；没有就 **自己加一行**（注意用你电脑上的路径，下面是例子）：

   ```text
   #用 Docker 跑 OpenClaw 时，容器里盘符是 /app/data（对应你电脑上的 openclaw/data）
   XHS_FACTORY_SAMPLES_PATH=/app/data/xhs-feed/samples.json
   ```
   改完 `xhs_factory.py` 等代码后，需要执行 **`docker compose build openclaw`** 再 **`docker compose up -d`**，容器里才会是新逻辑。

3. 保存。

### 第 4 步：让正在跑的程序重新读到配置

如果你是用 **`docker compose`** 跑的，在项目目录打开 PowerShell，执行：

```powershell
Set-Location -LiteralPath "D:\ai封装"
docker compose up -d
```

（不用懂含义：**相当于让后台服务再读一遍你的 `.env`**。）

### 第 5 步：看有没有用上「真样本」

再跑一遍生成小红书文案的流程（比如 `bench-hermes-xhs-sync.ps1`）。  
如果挖掘出来的内容里，标题 **像 `samples.json` 里的示例标题**，而不是一串 **`[sim-0]`** 开头的假数据，就说明 **你已经接通了**：  
**「抽屉里的文件 → 工厂」** 这条路是通的。

---

## 以后再接「真正的爬虫」

等你 MediaCrawler 会跑了，它导出的文件 **格式可能不一样**。那时可以：

- 把导出文件交给 **`scripts\export_to_xhs_feed.py`** 合并成 `samples.json`，  或者  
- 把导出内容 **整理成和 `samples.example.json` 差不多的样子**。

细节等你会用爬虫了再看 **`爬虫与工厂-对接清单与双文件夹打开步骤.md`**，不用一次看完。

---

## 还是懵的话

你可以先做到 **第 5 步**：只用 **`samples.json` 复制 example**，不配爬虫。  
只要这一步通了，你就已经理解 **「工厂只吃指定文件」** 这一件事；爬虫只是 **换一份更好的文件** 放进同一个抽屉。

有问题时，你可以直接说：卡在第几步、屏幕上看到什么字，再往下问。
