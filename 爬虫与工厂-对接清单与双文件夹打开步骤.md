# 爬虫与工厂对接 — 你要满足什么 & 双文件夹怎么打开

这份文档解决两件事：

1. **你在做 / 用 MediaCrawler 时，怎样才算「符合要求」**，工厂（`xhs_factory`）才能用上你的数据。  
2. **「一个窗口里同时看 ai封装 + 爬虫」** 在 Cursor 里具体要点哪里（不会用也没关系，文末有**更简单**的做法）。

---

## 一、先记住一句话

**工厂不依赖你的爬虫源码，只依赖磁盘上的 JSON 文件（或你自己提供的 HTTP 接口）。**  
你只要让文件 **格式对、路径对、编码对**，需求就满足了；不需要把爬虫写进 `ai封装` 仓库里。

---

## 二、爬虫侧「验收清单」（做完一条勾一条）

| 序号 | 要求 | 说明 |
|------|------|------|
| ① | **编码 UTF-8** | 保存 JSON / JSONL 时不要 ANSI / GBK，避免中文乱码。 |
| ② | **结构**：要么整文件是一个 **JSON 数组** `[{...},{...}]`，要么 **JSONL**（每行一个 `{...}`）。 | 与 `export_to_xhs_feed.py`、工厂读文件逻辑一致。 |
| ③ | **每条记录至少能凑出「标题 + 正文」** | 不要求字段名一模一样，工厂会映射（见下表）。至少要有能当标题、能当正文的文字，否则这条会被跳过。 |
| ④ | **点赞数（可选但建议有）** | 有则工厂用来做 `like_proxy`；没有会用默认值，仍能跑通。 |
| ⑤ | **文件放在工厂读得到的地方** | 常见：`D:\ai封装\openclaw\data\xhs-feed\samples.json`，并在 `.env` 里配置 `XHS_FACTORY_SAMPLES_PATH` 或 `XHS_FACTORY_FEED_DIR`（见 `.env.example`）。 |
| ⑥ | **改完样本后** | 若用 Docker跑 OpenClaw：保存文件后一般无需重建镜像；若改了 `.env`，执行一次 `docker compose up -d` 让容器重新加载环境。 |

### 字段对照（你导出里用这些名字之一即可）

| 含义 | 你可以用的字段名（任选一个存在的） |
|------|--------------------------------------|
| 标题 | `title`、`title_hint`、`note_title`；很短时也可用 `desc` / `description` |
| 正文/摘要 | `content`、`body_hint`、`note_text`、`desc`、`description` |
| 点赞 | `liked_count`、`likes`、`like_count`、`like_proxy` |
| 可选：结构/情绪标签 | `sop_tag` 或 `viral_sop`；`emotion_tag` 或 `target_emotion` |

**对照样例**：`openclaw/data/xhs-feed/samples.example.json`。

### 若 MediaCrawler 导出是「一整包嵌套 JSON」

先不要手改几百条：用仓库里的脚本合并并统一成数组（在 **`D:\ai封装`** 下打开 PowerShell）：

```powershell
Set-Location -LiteralPath "D:\ai封装"
python scripts\export_to_xhs_feed.py --in "你的导出文件或目录" --out "openclaw\data\xhs-feed\samples.json"
```

若本机命令是 `py` 而不是 `python`，把上面里的 `python` 改成 `py`。

## 三、「第一个方式」不会用：在 Cursor 里打开双文件夹（逐步）

目标：**一个窗口**里，左边文件树同时出现 **ai封装** 和 **MediaCrawler**，方便你一边看爬虫一边看工厂配置。

### 步骤 1：确定 MediaCrawler 在本机的路径

例如你 clone 到了 **`D:\MediaCrawler`**（文件夹必须真实存在，里面能看到 MediaCrawler 的 README 等）。

若你的路径不同，继续步骤 2。

### 步骤 2：改工作区文件里的路径

1. 在 Cursor 里打开：`D:\ai封装\dev-workspace.code-workspace` （这是一个纯文本 JSON，也可以用记事本打开。）
2. 找到 `"path": "../MediaCrawler"` 这一段。  
   - 若 MediaCrawler 在 **`D:\MediaCrawler`**（与 `D:\ai封装` **同级**），可改成绝对路径更不容易错，例如：

   ```json
   "path": "D:/MediaCrawler",
   ```

   注意：正斜杠 `/` 在 Windows 里也可以，且少踩转义坑。
3. 保存文件。

### 步骤 3：用「工作区」打开，而不是只打开单个文件夹

1. 在 Cursor 顶部菜单点 **文件 (File)**。  
2. 选 **从文件打开工作区…** / **Open Workspace from File…**（英文界面一般是这个）。  
3. 选中 **`D:\ai封装\dev-workspace.code-workspace`**，打开。  
4. 左侧 **资源管理器** 里应出现 **两个根**：`ai封装` 与 `MediaCrawler`。

若第二个根是空的或报错：说明步骤 2 里的路径写错或文件夹不存在，回到步骤 2 改成真实路径。

### 如果你找不到「从文件打开工作区」

- 仍可用 **两个 Cursor 窗口**：一个 **打开文件夹** `D:\ai封装`，另一个 **打开文件夹** `D:\MediaCrawler`。  
- **同步方式不变**：爬虫导出 → 写到 `D:\ai封装\openclaw\data\xhs-feed\`（或你配置的绝对路径）。**不需要**两个窗口「自动同步代码」。

---

## 四、怎样确认工厂真的用上了你的数据？

跑一条小红书工厂任务后，看挖掘阶段样本标题：若仍全是类似 **`[sim-0]`** 的模拟前缀，说明 **当前 OpenClaw 没读到你的 JSON**（检查 `.env` 路径、文件是否在挂载目录内、容器是否重启）。  
若标题变成你 JSON 里的真实标题，说明 **需求已满足**。
